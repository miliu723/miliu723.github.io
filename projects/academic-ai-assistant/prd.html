ScholarAI 产品需求文档（PRD）

一、产品概述

1.1 产品定位

ScholarAI 是一款面向科研人员的智能学术文献管理与分析工具，基于2026年最新大语言模型内核，聚焦解决科研人员文献检索效率低、解读难度大、管理碎片化等核心痛点，实现从文献检索、智能解读、对话问答到库管理的全流程学术工作流优化。

1.2 核心价值

  • 效率提升：将传统数天的科研调研周期缩短至数分钟
  • 深度解读：利用LLM对文献进行结构化、通俗化的智能分析
  • 全流程管理：覆盖文献检索、存储、解读、问答、导出的完整闭环
  • 个性化适配：支持多模型选择、总结风格自定义等个性化配置

1.3 目标用户

高校科研人员、硕博研究生、企业研发人员等需要高频检索、解读、管理学术文献的人群。

产品核心入口，承载搜索功能与核心能力展示，引导用户快速触达核心功能。

2.2 搜索结果页

2.2.1 功能定位

承接首页搜索请求，展示AI智能排序后的文献列表，支持多维度筛选、AI总结预览、快速收藏/查看详情等操作，是连接搜索入口与论文详情的核心页面。

2.2.2 核心需求

需求ID需求描述优先级交互细节
RES-001顶部搜索回显与导航最高1. 左侧返回按钮（点击跳转首页）+ 搜索框回显用户输入关键词<br>2. 搜索框支持重新输入、回车二次检索<br>3. 右侧展示检索耗时（例：检索耗时 0.8s）+ 结果总数（例：共找到 42 篇相关文献）<br>4. 整体固定悬浮，背景白色
RES-002多维度筛选栏高1. 横向滚动布局（移动端可滑动），包含以下筛选项：<br> - 时间维度：近1年 / 近3年 / 近5年 / 全部<br> - 期刊等级：SCI / 核心 / 普通<br> - 文献类型：期刊 / 会议 / 学位论文<br> - 排序方式：相关度（默认）/ 发表时间 / 被引量<br>2. 选中项高亮（蓝色背景+白色文字），未选中项hover浅灰背景<br>3. 筛选条件变更后，列表实时刷新（带加载动画）
RES-003AI智能排序提示中1. 筛选栏下方展示「AI已智能排序」提示条（蓝色左侧边框+AI图标）<br>2. 提示文字：“基于您的搜索意图，AI已优先展示高相关度、高影响力文献”<br>3. 点击提示条右侧「了解排序规则」可展开说明弹窗
RES-004文献列表卡片展示最高1. 每篇文献为独立白色卡片，hover态：阴影放大+轻微上浮<br>2. 卡片内容包含：<br> - 标题（加粗大号字体，点击跳转详情页）<br> - 元数据：作者 + 期刊 + 发表年份 + DOI（小号浅灰字体）<br> - AI核心总结：2行截断展示，前缀标注「AI总结：」（蓝色文字）<br> - 标签：核心贡献/实验数据/方法论（彩色小标签）<br> - 操作按钮：收藏（空心/实心星星图标）+ 查看详情（蓝色按钮）<br>3. 卡片底部展示被引量/下载量（带对应图标）<br>4. 列表支持懒加载（滚动到底部自动加载下一页）
RES-005空结果/无筛选结果提示中1. 无结果时展示：空状态插图 + 提示文字（例：未找到符合条件的文献）<br>2. 附带智能推荐关键词（例：您可能想找：Transformer 优化 / 大模型效率）<br>3. 推荐关键词点击可直接检索
RES-006分页/懒加载交互中1. 桌面端：列表展示10篇/页，底部分页器（上一页/页码/下一页）<br>2. 移动端：懒加载（滚动到底部自动加载，带加载动效）<br>3. 加载中展示骨架屏占位，避免页面抖动

2.2.3 视觉要求

  • 筛选栏与列表之间用浅灰分隔线区分，视觉层级清晰
  • 文献卡片采用白色背景+轻阴影，间距统一（16px），圆角8px
  • AI总结文本采用浅灰字体，与标题/元数据形成视觉区分
  • 标签颜色规范：核心贡献（蓝色）、实验数据（祖母绿）、方法论（琥珀色）
  • 加载动画采用蓝色渐变转圈，位置居中

三、非功能需求

3.1 性能要求

  • 页面加载时间≤2s（首屏），交互响应≤300ms
  • 支持1000+条文献列表的流畅滚动
  • 适配主流浏览器（Chrome/Firefox/Safari/Edge）及移动端浏览器

3.2 兼容性要求

  • 响应式适配：移动端（320px+）、平板（768px+）、桌面端（1200px+）
  • 适配不同屏幕密度（Retina/普通屏）
  • 图标采用矢量格式（Iconify），保证高清显示

3.3 体验要求

  • 所有交互元素（按钮/卡片/输入框）需有hover/active态反馈
  • 页面跳转采用平滑过渡，避免生硬刷新
  • 关键操作（如导出、删除）需有二次确认（后续迭代补充）
