# 会议记录 AI Agent 技术文档

## 技术栈

### 前端技术
- **框架**：React
- **语言**：TypeScript
- **状态管理**：Redux
- **UI 库**：Material-UI
- **样式**：CSS-in-JS (styled-components)

### 后端技术
- **语言**：Python
- **Web 框架**：FastAPI
- **数据库**：PostgreSQL
- **缓存**：Redis
- **认证**：JWT

### AI 技术
- **语音识别**：Whisper API
- **自然语言处理**：OpenAI GPT
- **文本嵌入**：Sentence-BERT
- **知识图谱**：Neo4j

### 核心模块

1. **语音处理模块**
   - 语音录制和上传
   - 语音转文本转换
   - 说话人识别

2. **文本分析模块**
   - 会议内容理解
   - 关键信息提取
   - 实体识别

3. **纪要生成模块**
   - 结构化模板生成
   - 内容组织和排版
   - 自动摘要

4. **存储和检索模块**
   - 会议记录存储
   - 全文搜索
   - 历史记录管理

## API 设计

### 主要端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/meetings` | POST | 创建新会议 |
| `/api/meetings/{id}` | GET | 获取会议详情 |
| `/api/meetings/{id}/record` | POST | 上传会议录音 |
| `/api/meetings/{id}/transcript` | GET | 获取会议转录文本 |
| `/api/meetings/{id}/summary` | GET | 获取会议纪要 |
| `/api/meetings` | GET | 列出所有会议 |
| `/api/meetings/{id}` | DELETE | 删除会议 |

### 请求/响应示例

#### 创建会议

**请求：**
```json
POST /api/meetings
{
  "title": "产品周会",
  "participants": ["张三", "李四", "王五"],
  "start_time": "2026-03-07T10:00:00Z",
  "end_time": "2026-03-07T11:00:00Z"
}
```

**响应：**
```json
{
  "id": "12345",
  "title": "产品周会",
  "participants": ["张三", "李四", "王五"],
  "start_time": "2026-03-07T10:00:00Z",
  "end_time": "2026-03-07T11:00:00Z",
  "status": "created"
}
```

## 部署方案

### 本地开发
- 使用 Docker Compose 启动所有服务
- 前端：`npm run dev`
- 后端：`python -m uvicorn app.main:app --reload`

### 生产部署
- 使用 Kubernetes 进行容器编排
- 前端部署到 CDN
- 后端服务部署到云服务器
- 数据库使用托管服务
- 定期备份数据
